# Second Brain Operating Model

## Thesis
This vault should behave like a maintained knowledge codebase. Raw inputs are preserved, derived pages are continuously improved, and useful answers are written back into the system instead of disappearing into chat history.

## Immediate Practices
- Capture new material in `raw/`.
- Ingest one source at a time when quality matters.
- Read `index.md` first before answering questions.
- Update topic pages whenever a source changes the standing understanding.
- Append every meaningful ingest or maintenance pass to `log.md`.

## Near-Term Build Order
1. Add the next real personal source.
2. Create the first area pages, likely `health`, `work`, and `learning`.
3. Start saving durable answers into `pages/queries/`.
4. Add lint passes once the page graph becomes denser.

## Decision
Use this source as the founding charter of the vault until a larger corpus exists.
